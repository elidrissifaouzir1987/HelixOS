Décisions ouvertes — recommandation du panel

Ces six forks changent la forme de tout ce qui est en aval. Chacune porte une reco argumentée ; ce sont elles qui appellent ton arbitrage avant d'écrire la moindre ligne.

Décision · verrou n°1archi B1 · red-team C1

La surface d'approbation « souveraine » (SPEC-003) est impossible telle qu'écrite

hermes-webui exécute l'agent in-process (il lit le SQLite) et n'expose aucune API pour y pousser un deep link. La frontière « pile fiable / non fiable » passe alors au milieu d'un seul processus. Un passkey prouve « l'humain a cliqué », pas « l'humain a vu la vérité ».

Reco

Micro-page servie par le noyau sur une origine distincte (certif dédié, frame-ancestors 'none') ; deep link livré hors-bande par ntfy, résumé + hash émis par le noyau, pas l'agent ; comparaison de hash pour les L2 ; cible = micro-PWA du noyau. Abandonner « zéro fork » comme dogme — c'est un slogan, pas une contrainte tenable.

Décisionagent langage

Langage du noyau — Rust cœur + sidecar C#/.NET

Go est éliminé : pas de révocation mTLS dans la stdlib, VSS backup-context impossible via WMI — deux invariants constitutionnels à recoder. C# seul gagne l'interop brute mais seulement en JIT, ce qui sacrifie le binaire unique et gonfle la surface d'attaque du composant souverain.

Reco

Cœur Rust (rustls révocation CRL native, webauthn-rs, #![forbid(unsafe_code)], binaire statique) + sidecar C#/.NET JIT pour VSS/COM — « une main, pas une tête » : ni policy ni HITL, verbes typés en localhost. Exige un amendement du Principe VIII (le noyau n'est plus « un seul binaire »). Plan B : monolithe C#/.NET JIT si le split coûte trop cher au solo.

DécisionWindows-frontier M1 · day-2 §3.1

Runtime conteneur — abandonner Docker Desktop

Double motif convergent. Isolation : Docker Desktop réintroduit des ponts hôte massifs (\\.\pipe\docker_engine, bind-mounts Windows automatiques, moteur partagé cross-distro) que le durcissement WSL2 ne peut pas fermer. Disponibilité : il ne démarre pas sans session ouverte — fatal pour une machine qui reboote seule la nuit.

Reco

dockerd natif dans la distro WSL2 durcie + systemd (ou Podman rootless). Interdire et tester : montage docker.sock, network_mode: host, privileged, pid/ipc: host. Reformuler la promesse du harness : « prouve que les réglages tiennent », jamais « prouve l'inévasibilité ».

Décisionarchi M1 / B3 / M2

Rôle et périmètre de Graphify

Outil fondamentalement codebase-first, 3 mois, pré-1.0, branche v8, churn massif. Son watch ne reconstruit pas le markdown ; l'extraction images/docs est un vision-LLM (cloud sauf Ollama).

Reco

MVP = indexation code + markdown + transcription (vraiment local, sans GPU) ; images/PDF-vision = extension explicite « Ollama-GPU ou cloud par exception ». Fraîcheur déclenchée par le noyau à chaque mutation validée. Pinner la version, limites cgroup, et sortir obsidian.* du cœur souverain (Obsidian n'est pas un concept OS).

Décision · doctrinered-team C2 / C3

Combler le trou topologie → rayon de souffle

Un agent compromis (prompt injection, non résolu en 2026) détient déjà le credential légitime du noyau et émet des intentions typées parfaitement valides. Le test #6 « donnée ≠ instruction » est un pass/fail non falsifiable — du théâtre.

Reco

Écrire noir sur blanc : l'agent est présumé compromissible, la défense est en aval. Deny-list secrets sur read_file (*.env, .ssh/, .hermes/) → L2 même en lecture ; budgets d'exfiltration (volume + nb fichiers) ; credential du noyau illisible par toute intention ; taint tracking (contenu non fiable lu ce tour → +1 cran HITL) ; test #6 → éval adversariale continue.

Décision · matérielday-2 §3

Réalité v1 : un poste, pas un serveur

Le cluster WoL est L2 / Tailscale est L3 (réveil impossible via le tailnet), expiry de clé Tailscale à J+180 (lockout silencieux), reboots Windows Update, et cold-start Docker sans session — tous pointent au même endroit.

Reco

Un petit relais Linux toujours allumé (Pi / mini-PC) sur le LAN : porte le magic packet WoL, le point d'entrée Tailscale stable, le healthcheck externe. C'est l'extension « machine dédiée » de la roadmap, avancée en P1 de fait. À défaut : workstation qui ne dort jamais + expiry de clé désactivé (tag:server).

A

Défauts confirmés — à corriger dans les docs

Non débattables : chaque point croise une affirmation des docs avec un fait de vérité-terrain qui la contredit. Ce sont exactement les bugs invisibles au mock que tes tests live rattrapent d'habitude — sauf qu'ici il n'y a pas encore de code.

CorrigerMajeurarchi B3 · day-2 §5

« GPU jamais requis / OCR-captions légers sur CPU » — faux

Graphify n'a aucun OCR/caption local : images, PDF et docs passent par un vision-LLM (cloud sauf Ollama). Le vrai axe est local vs cloud, pas CPU vs GPU. Seuls le code (tree-sitter) et la transcription (faster-whisper int8, confirmé verbatim) sont réellement locaux-sans-GPU.

→ Supprimer « OCR/captions légers sur CPU » des trois documents ; reclasser la couche connaissance en « local (code+transcription) » vs « LLM requis (le reste) ».

CorrigerMajeurarchi M1

« Fraîcheur < 1 min sur le vault » — impossible

Le watch de Graphify ne reconstruit pas le markdown : il pose un flag needs_update + notification (l'extraction LLM est payante). Atteindre <1 min exigerait un orchestrateur externe qui ré-extrait à chaque édition, avec coût LLM par frappe.

→ NFR redéfini : <1 min pour le code ; vault rafraîchi par le noyau à chaque mutation validée (il est déjà le point de passage obligé).

CorrigerMajeurarchi M2 · red-team M2

Secrets .env en clair — viole le Principe I

L'état Hermes est un unique bind-mount ~/.hermes contenant les clés en clair (CVE-2026-53870 : 0644 en <0.16.0). Or le Principe I exige « les secrets hors runtime agent ». Le design viole son propre invariant, et c'est une précondition d'exfiltration (voir décision B5).

→ Exiger Hermes ≥ 0.16.0 comme contrainte versionnée ; secrets 0600 / externalisés ; clé du noyau cloisonnée hors .env.

CorrigerMineurarchi M5 · day-2 §7.5

« WhatsApp/Twilio » — triple erreur factuelle

Le WhatsApp d'Hermes = pont Baileys (émulation non officielle, risque de ban du numéro) ; Twilio est la voie du canal SMS ; le skill complet est une issue ouverte. Faire reposer le « secours » là-dessus est fragile.

→ Canal de notification canonique = ntfy ; WhatsApp = commodité optionnelle documentée non fiable.

CorrigerMineurarchi M4 · red-team C2

subagent_id déclaratif — zéro valeur de sécurité

Le champ est fourni par l'agent — précisément le composant non fiable — et non authentifié. Un agent compromis y met n'importe quoi. « Préserve la granularité de l'audit » surpromet : un champ contrôlé par l'attaquant n'est pas de la forensique.

→ Requalifier en « hint debug/coûts, sans valeur de sécurité ». Traçabilité fiable = credential mTLS + plan signé.

CorrigerMineurarchi M2 / m4

Portabilité affirmée mais jamais exercée

« Portable dès le 1er jour » est testé sur un seul driver. Un contrat n'est prouvé portable qu'au 2e driver — et deux abstractions fuient déjà l'OS-spécifique : la classe de rollback (dépend du filesystem) et obsidian.* (produit tiers, pas concept OS).

→ Rétrograder en « architecture prête pour la portabilité » ; retirer de la colonne « Robuste » tant qu'un 2e driver ne l'a pas prouvée.

CorrigerMineurarchi m5 / m6

Graphify « en conteneur » = serving MCP seulement

Le Dockerfile officiel conteneurise le serveur MCP uniquement ; pas d'image préconstruite, et l'extraction/watch en conteneur n'est pas le scénario documenté.

→ Deux conteneurs (serving :ro + extracteur custom en écriture) + limites cgroup CPU/RAM pour rendre le test 8 vrai par construction.

CorrigerMineurarchi m3

Matrice test ↔ SPEC désalignée

Le test 5 (note Obsidian) est attribué à SPEC-003, mais la capacité obsidian.* n'est livrée qu'en SPEC-004 → critère de sortie infaisable. Le test 12 (TOCTOU) est compté double (002 + 003).

→ §9 = source unique ; chaque test appartient au lot qui livre sa capacité ; les recouvrements deviennent « re-vérifié », pas « possédé ».

CorrigerMineurarchi m1

Constitution non ratifiée

Ratifiée le : [DATE] alors que le texte « gouverne toutes les specs ». Formellement, rien n'est encore gouverné. Le choix Rust/Go est aussi laissé flottant jusqu'à SPEC-002 alors qu'il conditionne SPEC-001.

→ Ratifier avec une vraie date + trancher le langage (décision B2) avant de démarrer.

E

Vérité-terrain Windows — ce qui tient, ce qui ment

La cible v1 est une workstation Windows : le noyau natif pilote l'hôte via VSS, la recherche de fichiers et PowerShell, et sert une page WebAuthn sur le tailnet. Quatre de ces promesses ont été passées aux sources primaires Microsoft.

Confirmé · bonne nouvelleagent WebAuthn

WebAuthn / passkey sur le tailnet — solide

Un service Windows natif peut servir la page d'approbation et vérifier WebAuthn correctement. Le crux est résolu : le RP ID = un nom MagicDNS (helix.<tailnet>.ts.net) marche parce que ts.net est sur la Public Suffix List, donc <tailnet>.ts.net est un eTLD+1 propre — isolation par-tailnet gratuite. tailscale cert fournit un certif Let's Encrypt (secure context) ; servir depuis l'origine du noyau (same-origin) évite tout le bazar cross-origin iframe.

Attestation none = correct pour un usage perso. Jamais l'IP 100.x (ni RP ID valide, ni certifiable).

Fragileagent langage · day-2 §7.4

VSS pour le rollback auto — la taxonomie est à l'envers

VSS est par-volume (jamais par-fichier), impose un gel writer de plusieurs secondes, se perd sous forte I/O, échoue disque plein, et son chemin de création « facile » (vssadmin/WMI) est server-only ou en cours de suppression — seul le COM IVssBackupComponents est supporté client, absent de win32metadata (FFI Rust douloureux) ; AlphaVSS (C#) est le chemin mûr mais C++/CLI → incompatible AOT (d'où le sidecar JIT).

→ Inverser la taxonomie : compensation (copie-aside + ReplaceFile atomique, déterministe, tout filesystem, sans élévation) devient la classe garantie par défaut ; auto (VSS) l'exception opportuniste derrière un probe (NTFS fixe + writers sains + espace + élévation), un snapshot par lot jamais par fichier.

Nuancéagents Everything / Windows Search

Recherche de fichiers — dépendance à requalifier

Everything dépend de l'app + service tiers + index USN. L'alternative native Windows Search (OLE DB Search.CollatorDSO, zéro install) est conditionnellement utilisable depuis un service session-0 : pas whole-disk par défaut, asynchrone (pas live), et la visibilité des résultats sous l'identité LocalSystem doit être prouvée par un spike.

→ Traiter la recherche comme capacité de driver remplaçable, pas dépendance dure ; valider par un spike avant de committer.

NuancéWindows-frontier B

Firewall Hyper-V « un seul port » — vrai pour le routé, pas le loopback

On peut bien passer WSL en DefaultOutboundAction=Block + une règle d'autorisation. Mais le loopback host↔guest est gouverné par un toggle global LoopbackEnabled, pas par port. « Exactement un port, loopback compris » n'est pas configurable finement ; il faut binder l'endpoint sur la gateway WSL (routée), pas sur 127.0.0.1.

→ mTLS par cert client par conteneur (l'identité vient du cert, pas du réseau) reste la seule voie viable vu l'override réseau de Docker Desktop.

D

Ce qui survit comme sain

Le stress-test n'invalide pas le projet : la colonne vertébrale est bonne. Ces sept piliers tiennent — la plupart moyennant les correctifs ci-dessus, aucun n'est à jeter.

  • Doctrine « sécurité = topologie » comme première ligne — à condition d'ajouter le bornage du rayon de souffle (B5).
  • Plan signé (hash / TTL / usage unique) + HITL gradué L0/L1/L2 + audit append-only + idempotence = bonne colonne vertébrale.
  • Noyau portier unique à intentions typées — la bonne abstraction, à compléter par le blast-radius.
  • Taxonomie de rollback honnête (auto / compensation / irréversible, dégradable jamais surdéclarée) — à étendre au cas « disque plein ».
  • CPU-first pour la transcription (faster-whisper int8 confirmé), GPU opportuniste — le seul mode dégradé déjà bien traité.
  • Tailscale deny-by-default garde 8642/9119 non exposés — bon ; chiffrer le state file, activer Tailnet Lock.
  • Séparation « Tailscale protège l'accès, le noyau protège les actions » — la bonne ligne de partage.
